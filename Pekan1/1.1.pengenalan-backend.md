# Pengenalan Backend Development  
### Memahami Otak di Balik Aplikasi

## Backend: Otak dan Dapur Pacu Sebuah Aplikasi

**Definisi Sederhana**  
Bagian dari aplikasi yang **tidak terlihat oleh pengguna** (di balik layar). Backend bekerja di **server** untuk menjalankan logika dan mengelola data.

**Peran Utama**  
Mengelola data, logika bisnis, dan komunikasi dengan database untuk mendukung semua fungsi aplikasi.

**Fungsi Kunci**
- Menyimpan dan mengambil data dari Database
- Menjalankan logika dan aturan bisnis
- Memastikan keamanan dan otentikasi pengguna

## Backend Bekerja di Sekitar Kita

| Contoh Sehari-hari      | Apa yang Dilakukan Backend |
|-------------------------|-----------------------------|
| **Login Akun**          | Memeriksa username & password cocok dengan data di database |
| **Pencarian Produk**    | Mencari jutaan data produk dan menampilkan hasil relevan dengan cepat |
| **Transaksi Online**    | Memproses pembayaran, mengurangi stok, mencatat riwayat pembelian |
| **Mengirim Pesan**      | Menerima pesan dan mengirimkannya ke penerima dengan aman & cepat |

## Frontend vs Backend: Siapa Melakukan Apa?

| Fitur       | Frontend (Sisi Klien)               | Backend (Sisi Server)                  |
|-------------|-------------------------------------|----------------------------------------|
| **Peran**   | Tampilan, Interaksi Pengguna        | Logika Bisnis, Data, Keamanan          |
| **Fokus**   | Apa yang dilihat pengguna (UI/UX)   | Bagaimana aplikasi bekerja             |
| **Contoh**  | Tombol, Warna, Layout, Animasi     | Database, API, Otentikasi              |
| **Lokasi**  | Browser Pengguna (HP/Laptop)        | Server Jarak Jauh (Cloud)              |

## HTTP: Bahasa Percakapan Internet

**Hypertext Transfer Protocol** â€“ aturan dasar komunikasi klien-server di internet.

**Komponen Utama**
- **Method**: Jenis aksi (`GET`, `POST`, `PUT`, `DELETE`)
- **Header**: Informasi tambahan (tipe data, autentikasi, dll)
)
- **Body**: Isi data yang dikirim/diterima (misalnya data form login)

### HTTP Methods (CRUD)

| Method | Operasi  | Contoh                              |
|--------|----------|-------------------------------------|
| GET    | Read     | Membuka halaman produk              |
| POST   | Create   | Mengisi formulir pendaftaran        |
| PUT    | Update   | Mengganti semua detail profil       |
| DELETE | Delete   | Menghapus postingan                 |

> Dasar dari konsep **CRUD** (Create, Read, Update, Delete)

### Status Code: Laporan Hasil Komunikasi

| Kode | Arti                     | Penjelasan                                      |
|------|--------------------------|-------------------------------------------------|
| 200  | OK                       | Permintaan berhasil                             |
| 201  | Created                  | Data baru berhasil dibuat                       |
| 400  | Bad Request              | Data yang dikirim salah format                  |
| 401  | Unauthorized             | Belum login                                     |
| 403  | Forbidden                | Login tapi tidak punya izin                     |
| 404  | Not Found                | Halaman/data tidak ada                          |
| 500  | Internal Server Error    | Ada masalah di server                           |
