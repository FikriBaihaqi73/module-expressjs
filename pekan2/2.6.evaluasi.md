### 1. Lanjutan Project
- Gunakan repository project **Perpustakaan** yang telah dibuat pada evaluasi Pekan 1.
- Refactor kode yang ada (sebelumnya mungkin menggunakan in-memory array atau driver database dasar) untuk menggunakan **Prisma ORM** dengan database PostgreSQL.

### 2. Aturan Database & Schema
- **Integrasi Prisma:** Seluruh operasi database wajib menggunakan Prisma Client.
- **Relasi 1 : N (One-to-Many):**
  - **Wajib** ada minimal satu relasi One-to-Many.
  - **Instruksi Khusus:** Tambahkan 2 resource/tabel baru yang berelasi One-to-Many.
  - *Catatan:* Jika resource/entitas yang sudah ada di Pekan 1 (misal `Book` dan `Category`, atau `Member` dan `Loan`) sudah mendukung atau bisa dimodifikasi untuk relasi ini, maka Anda tidak perlu membuat resource baru yang benar-benar terpisah. Cukup implementasikan relasi tersebut pada resource yang ada.
- **Best Practices:**
  - Gunakan **UUID** sebagai Primary Key untuk semua tabel.
  - Implementasikan **Soft Delete** (`deletedAt`) pada schema database dan logika query.

### 3. Fitur API & Endpoint
Setiap resource utama (yang didefinisikan dalam tugas ini) wajib memiliki **6 Endpoint** berikut:

1.  **GET** `/resource` (Mendapatkan semua data dengan pagination opsional).
2.  **GET** `/resource/:id` (Mendapatkan detail data berdasarkan ID).
3.  **POST** `/resource` (Membuat data baru dengan validasi).
4.  **PUT/PATCH** `/resource/:id` (Update data dengan validasi).
5.  **DELETE** `/resource/:id` (Implementasi **Soft Delete**, bukan menghapus permanen dari DB).
6.  **SEARCH** `/resource/search` (Mencari data berdasarkan keyword tertentu, misal judul buku atau nama penulis).

### 4. Implementasi Teknis & Best Practices
- **Validation:** Lakukan validasi input request (body/params) sebelum masuk ke database (bisa menggunakan `zod`, `joi`, atau `express-validator`).
- **Error Handling:** Terapkan **Centralized Error Handling** (Middleware) untuk menangani error, termasuk spesifik **Prisma Error Handling** (contoh: Unique Constraint Violation, Record Not Found).
- **Struktur Folder:** Gunakan arsitektur MVC atau modular yang rapi.
