## 2. Mini Project Evaluasi: "Keranjang Belanja (Cart) & Wishlist"

**Skenario:**
Sebelum user melakukan Checkout (yang sudah dipelajari di Hari 13), biasanya user ingin menampung barang dulu di **Keranjang (Cart)** atau menyimpan barang impian di **Wishlist**.

**Tugas:** Kembangkan project `e-commerce-api` yang sudah ada.

### Entitas (Schema Prisma)
1.  **Cart** & **CartItem**
    - 1 User punya 1 Cart.
    - 1 Cart punya Banyak CartItem (Product + Qty).
2.  **Wishlist**
    - 1 User punya Banyak Wishlist (Product).

### Fitur Wajib
1.  **Auth:** Hanya user yang Login yang punya Cart/Wishlist.
2.  **Cart Logic:**
    - `POST /api/cart`: Tambah barang ke keranjang. Jika barang sudah ada, update Qty-nya.
    - `GET /api/cart`: Lihat isi keranjang.
    - `DELETE /api/cart/:itemId`: Hapus barang dari keranjang.
3.  **Wishlist Logic:**
    - User bisa simpan produk ke wishlist.
    - User bisa lihat daftar wishlist mereka.
4.  **Architecture:** WAJIB menggunakan **Repository-Service Pattern** (Lanjutan materi Pekan 3 Hari 17).

### Kriteria Penilaian
- **Auth:** Token JWT berfungsi untuk membedakan "Keranjang Siapa Ini?".
- **Logic:** Tidak boleh ada barang duplikat di Wishlist (Unique constraint).
- **Code:** Struktur folder rapi (Controller-Service-Repo).
